Networking Comprehensive Guide

Detailed explanation of networking topics, models, and transmission types.

Unit 1: Introduction to Networking & Network Models

Basics of Networking

A computer network is a group of interconnected computers and devices that can communicate with one another to share resources, data, and applications.

Advantages:

They allow for shared resources (like printers), centralized data management, rapid communication and collaboration, and access to remote information.

Disadvantages:

Networks require a high initial setup cost, introduce security risks and vulnerabilities to viruses/malware, demand specialized maintenance, and create a dependency on central infrastructure.

Network Types (LAN, MAN, WAN)

  • LAN (Local Area Network): Designed to operate within a limited geographic area (like an office or building). They allow multi-access to high-bandwidth media, are privately administered, and provide full-time connectivity to physically adjacent devices.
  • WAN (Wide Area Network): Operates over massive geographic areas using telecommunications carriers. They allow access over serial interfaces operating at lower speeds and can connect devices globally.
  • MAN (Metropolitan Area Network): A network that spans a physical area larger than a LAN but smaller than a WAN, such as a city or a large university campus. It is typically used to connect several LANs together to form a larger network.

Network Topologies

Topology defines the physical and logical structure of how network components are interconnected.

Bus Topology: All stations connect to a single backbone cable. It is cost-effective and failure in one node does not affect the rest, but if the main cable fails, the entire network is disrupted. It also suffers from signal interference if two nodes transmit simultaneously.

Ring Topology: Devices connect to exactly two neighboring devices, forming a ring. Transmissions are usually unidirectional and pass through nodes sequentially. Because data travels through many nodes, repeaters are used to prevent data loss.

Star Topology: All devices connect to a central hub or switch. It is easily expandable and failure of one cable only affects one node. However, if the central hub goes down, the entire network fails.

Tree Topology: Combines bus and star topologies in a parent-child hierarchy with a root node. It is easily expandable and manageable, but relies heavily on the main bus cable.

Mesh Topology: Features multiple redundant paths between computers. It is highly reliable because a broken link doesn't stop communication, but it is expensive and incredibly complex to manage.

Hybrid Topology: A combination of two or more different topologies. It is flexible and reliable, but carries a high infrastructure cost and complex design.

OSI and TCP/IP Models

graph BT L7[7. Application] --- L6[6. Presentation] L6 --- L5[5. Session] L5 --- L4[4. Transport] L4 --- L3[3. Network] L3 --- L2[2. Data Link] L2 --- L1[1. Physical]

OSI Model: Divides network communication into 7 distinct layers.

  1. Physical: Handles binary transmission, voltage levels, and physical media.
  2. Data Link: Breaks data into frames, handles MAC addressing, and provides error and flow control.
  3. Network: Routes packets through the network and defines logical addressing (IP).
  4. Transport: Ensures reliable, error-free, end-to-end message transmission (TCP/UDP).
  5. Session: Establishes, maintains, and synchronizes dialogs between systems.
  6. Presentation: Handles data formatting, compression, and encryption.
  7. Application: Interacts directly with software applications.

TCP/IP Model: A streamlined 4-layer conceptual model consisting of Application, Transport, Internet (Network), and Network Access (Data Link/Physical) layers.

Overview of Networking Devices

  • Repeater: Operates at the physical layer to regenerate weak signals before they corrupt.
  • Hubs: Connect several computers. They cannot filter data and instead broadcast packets to all connected devices, making them inefficient. Active hubs amplify signals, while passive hubs do not.
  • Switches: Data link layer devices that perform error checking before selectively forwarding data only to the correct port.
  • Routers: Network layer devices that connect different networks (LANs to WANs) and route data packets dynamically using a routing table based on IP addresses.

Network Protocols and Standards

Protocols: A set of rules covering data formatting, transmission, error handling, and security. Key elements include syntax (format), semantics (meaning), and timing.

Standards: Accepted design rules that ensure devices from different manufacturers are compatible. They can be De Facto (widely used but not officially approved) or De Jure (officially approved by bodies like IEEE).

Unit 2: Data Communication & Transmission Techniques

Analog vs. Digital Signals

Analog Signal (Continuous)

Digital Signal (Discrete)

Analog Signal: A continuous, time-varying signal with an infinite number of values within a specific range. They preserve original signals but produce high noise.

Digital Signal: Represents data as a sequence of discrete, discontinuous values (1s and 0s). They do not produce noise and are easier to analyze and store.

Data Encoding Techniques

Line coding converts digital data into digital signals.

Unipolar NRZ (Non-Return-to-Zero) Signal

Unipolar: Uses only one polarity (e.g., positive voltage for 1, zero voltage for 0). It is problematic because it creates a DC component.

Polar & Bipolar: Bipolar uses three voltage levels (positive, negative, zero) where alternate 1s use opposite polarities to prevent DC components.

Manchester Encoding: Uses an inversion at the middle of each bit interval (e.g., negative-to-positive is binary 1).

Differential Manchester: Uses a transition to represent binary 0 and no transition for binary 1.

Transmission Impairments

As signals travel, they degrade due to three main causes:

  1. Attenuation: The loss of signal energy over distance, requiring amplifiers.
  2. Distortion: Changes in signal shape because different frequencies travel at different speeds and arrive out of phase.
  3. Noise: Unwanted signals including thermal noise, induced noise from appliances, crosstalk between wires, and impulse spikes.

Multiplexing Methods

Allows multiple signals to share a single medium.

  • FDM (Frequency-Division Multiplexing): An analog technique dividing available bandwidth into distinct frequency channels.
  • TDM (Time-Division Multiplexing): A digital technique where signals share the same frequency but take turns in time slots. It can be Synchronous (fixed slots) or Asynchronous/Statistical (dynamic slots based on need).
  • WDM (Wavelength-Division Multiplexing): Combines multiple optical signals on a single fiber using different colors (wavelengths) of light. Includes CWDM and DWDM.

Switching Techniques

Circuit Switching: Establishes a dedicated, unbroken physical path between the sender and receiver. It guarantees capacity but is expensive and has a long connection setup time.

Message Switching: A store-and-forward method where the entire message travels node-to-node. It reduces congestion but requires massive storage disks.

Packet Switching: Messages are broken into small packets. In the Datagram approach, packets take independent routes and are reordered at the destination. In the Virtual Circuit approach, a preplanned logical route is established first.

Transmission Media Types

Guided (Wired): Twisted-Pair cable (UTP/STP), Coaxial cable, and Fiber-Optic cable (uses light pulses through a glass/plastic core and cladding).

Unguided (Wireless): Transport electromagnetic waves without physical conductors. Includes Radio waves (omnidirectional), Microwaves (unidirectional, line-of-sight), and Infrared (short-range, blocked by walls).

Unit 3: Data Link Control & MAC Protocols

Error Detection and Correction Techniques

Because data can be corrupted, redundancy bits are added.

Detection: VRC (Vertical Redundancy Check/Parity), LRC (Longitudinal Redundancy Check), Checksum, and CRC (Cyclic Redundancy Check).

Correction: The Hamming Code is used to pinpoint the exact location of a damaged bit so the receiver can reverse it.

Flow Control Mechanisms

Ensures a fast sender does not overwhelm a slow receiver.

Stop-and-Wait: The source transmits one frame and waits for an acknowledgment (ACK) before sending the next.

Sliding Window: Allows multiple numbered frames to be transmitted without an immediate ACK. The receiver uses a buffer window to track them.

MAC Addressing and ARQ Protocols

MAC Addressing: Every device's Network Interface Card (NIC) has a 6-byte (48-bit) physical address written in hexadecimal. It can be Unicast, Multicast, or Broadcast.

ARQ (Automatic Repeat Request): Combines error detection with retransmission.

  • Stop-and-Wait ARQ: If a frame or ACK is lost/damaged, the sender's timer expires, and it retransmits the frame.
  • Go-Back-N ARQ: If one frame fails, the receiver rejects it, and the sender must retransmit the failed frame and all subsequent frames in the window.
  • Selective Reject ARQ: Only the specific rejected frame is retransmitted, while the receiver buffers the subsequent good frames.

Ethernet Standards

Defined by IEEE 802.3. It has evolved from Standard Ethernet (10 Mbps) to Fast Ethernet (100 Mbps), Gigabit Ethernet (1 Gbps), and Ten-Gigabit Ethernet (10 Gbps). The Ethernet frame consists of fields including the Preamble, SFD, Destination/Source Address, Length/Type, Data payload, and CRC.

Wireless LAN Protocols

Wi-Fi (IEEE 802.11): Operates on 2.4 GHz and 5 GHz frequency bands. It supports infrastructure and ad-hoc networks but is susceptible to interference.

Bluetooth (IEEE 802.15.1): A short-range standard for ad-hoc networks, utilizing UHF radio waves. Devices connect in a "Piconet" or "Scatternet" structure. Includes a Low Energy (BLE) version.

Unit 4: Network Communication Protocols

Application Protocols

HTTP/HTTPS (Hypertext Transfer Protocol): The sources note this is used for "Web data transfer". HTTP is the foundation of data communication on the World Wide Web. It dictates how messages are formatted and transmitted, and how web browsers and servers should respond to commands. It operates over TCP port 80. HTTPS is the secure version, utilizing TLS/SSL encryption over TCP port 443 to protect data privacy.

FTP (File Transfer Protocol): The sources state this is used for "File sharing". FTP is a standard network protocol used to transfer computer files between a client and a server. It uses separate control and data connections, allowing a user to authenticate, list directories, and download/upload files reliably.

SMTP (Simple Mail Transfer Protocol): The sources state this is used for "Email sending". SMTP is the industry standard for sending emails across the internet. Mail servers use it to push messages to other servers, routing the email toward its destination. It relies heavily on DNS to find the mail exchanger (MX) records of the receiving domain.

DNS (Domain Name System): The sources state it "Converts URLs to IP addresses". DNS acts as the phonebook of the internet. Because humans read domain names (like www.google.com) but computers route data using IP addresses (like 192.168.1.1), DNS servers query a distributed database to translate the human-readable hostname into the machine-readable IP address required to load internet resources.

Transport Layer Protocols

TCP (Transmission Control Protocol): The sources note it provides "Reliable delivery with error checking" and "Management of end-to-end message transmission". TCP is a connection-oriented protocol. Before data is sent, it establishes a dedicated connection via a "three-way handshake." It ensures that all packets arrive intact and in the exact order they were sent by utilizing sequence numbers and requiring acknowledgments. If a packet is lost, TCP automatically requests a retransmission, making it highly reliable.

UDP (User Datagram Protocol): The sources note it provides "End-to-end delivery". UDP is a connectionless protocol. Unlike TCP, it does not establish a connection before sending data, nor does it check for lost packets or guarantee delivery order. Because it avoids this overhead, it is much faster than TCP and is primarily used in time-sensitive applications like live video streaming, online gaming, and VoIP calls, where occasional packet loss is acceptable.

Part A – Very Short Questions (1 Mark)

1. Define DNS.
DNS (Domain Name System) is an application layer protocol that converts human-readable URLs (web addresses) into machine-readable IP addresses.

2. What does IP stand for?
IP stands for Internet Protocol. It is responsible for logical packet addressing and routing across networks.

3. Which layer of OSI is responsible for encryption?
The Presentation Layer (Layer 6) of the OSI model is responsible for data representation, data compression, and data encryption (cryptography).

4. What is the main function of a router?
A router is a network layer device that connects different networks together and dynamically routes data packets based on their IP addresses.

5. Name one wireless protocol.
Wi-Fi (IEEE 802.11) and Bluetooth (IEEE 802.15.1) are common wireless protocols used for short-range and local network data communication.

6. Write the full form of SMTP.
SMTP stands for Simple Mail Transfer Protocol. It is an application-level protocol used for email sending.

7. What is the function of a repeater?
A repeater operates at the physical layer to receive a weak signal, copy it bit by bit, and regenerate it at its original strength before passing it on.

8. Name the topmost layer in the TCP/IP model.
The Application Layer is the topmost conceptual layer in the TCP/IP model.

Part B – Short Answer Questions (2 Marks)

9. Define a computer network.
A computer network is a group of interconnected computers and devices linked to each other. It enables these devices to communicate, exchange data, and share physical or logical resources efficiently.

10. List any three advantages of computer networks.
Three major advantages of computer networks include the ability to share resources and ease of access, efficient person-to-person communication and collaboration, and centralized data management and storage.

11. What is the main disadvantage of using a centralized network?
A major disadvantage is the dependency on a single central server or hub, such as in a star topology. If this central infrastructure fails or goes offline, the entire network is disrupted and communication stops.

12. Define LAN, MAN, and WAN.
A LAN (Local Area Network) operates within a limited geographic area to connect physically adjacent devices. A WAN (Wide Area Network) connects devices over massive geographic areas, even globally, using telecommunication carriers. (Standard knowledge: A MAN, or Metropolitan Area Network, spans an intermediate area like a city or large campus).

13. What is network topology? Name any four types.
Network topology defines the physical or logical structure of how all the components and nodes in a network are interconnected. Four common types of network topologies are Bus, Star, Ring, and Mesh topologies.

14. Define a protocol and give two examples.
A protocol is a set of rules that governs data transmission, covering aspects like data formatting, timing, error handling, and security. Two examples are TCP (for reliable delivery) and HTTP (for web data transfer).

15. What is a standard? Differentiate between De Facto and De Jure standards.
A standard is a set of accepted design rules that ensure compatibility between different systems and devices. "De Facto" standards are widely used but not officially approved by a body (e.g., Apple ecosystem), whereas "De Jure" standards are officially approved by organizations like ISO or IEEE.

16. Name any four networking devices and their functions.
A Hub broadcasts data blindly to all ports. A Switch selectively forwards packets error-free to the correct port. A Router connects networks and routes packets dynamically based on IP addresses. A Repeater regenerates weak physical signals to extend their range.

17. Write any two differences between OSI and TCP/IP models.
The OSI reference model rigidly divides network communication into seven distinct layers, whereas the TCP/IP conceptual model streamlines this into four layers. Furthermore, OSI is primarily a theoretical reference model, while TCP/IP is the practical protocol suite that runs the internet.

18. What is DHCP and why is it used?
DHCP stands for Dynamic Host Configuration Protocol. It is used to automatically and dynamically assign IP addresses to devices from a pool, making network management efficient and cost-effective.

Unit 2: Data Communication and Transmission Techniques

Part A – Very Short Questions (1 Mark)

1. Define Analog Signal.
An analog signal is a continuous, time-varying electrical current that is bound to a range and possesses an infinite number of values within that continuous range.

2. Define Digital Signal.
A digital signal represents data as a sequence of discrete, discontinuous values, taking on only one value from a finite set at any given time.

3. What is data encoding?
Data encoding (or line coding) is the process of converting internal digital data, represented as 1s and 0s, into digital signals for transmission across a medium.

4. Expand NRZ.
NRZ stands for Non-Return to Zero. It is a polar encoding technique where the voltage level does not return to zero during a bit interval.

5. Write the full form of FDM.
FDM stands for Frequency-Division Multiplexing. It is an analog technique that subdivides the available bandwidth of a medium into several frequency channels.

6. Define attenuation.
Attenuation is a transmission impairment causing the loss of signal energy as it travels through overcoming the resistance of the medium, resulting in a weaker signal.

7. Which encoding technique uses transitions in the middle of a bit interval?
Both Manchester Encoding and Differential Manchester Encoding use transitions (or inversions) directly at the middle of each bit interval for synchronization.

8. Name any two guided transmission media.
Two examples of guided (wired) transmission media are Twisted-Pair cables (such as UTP or STP) and Coaxial cables.

Part B – Short Answer Questions (2 Marks)

9. List any two differences between analog and digital signals.
Analog signals provide a continuous representation of data and preserve original signal shapes, but they produce too much noise. Digital signals provide a discontinuous representation, do not produce noise, and are much easier to analyze and store.

10. What is the function of Manchester Encoding?
Manchester encoding represents binary 1 as a negative-to-positive transition and binary 0 as a positive-to-negative transition. It uses an inversion exactly at the middle of each bit interval, which provides built-in clock synchronization for the receiver.

11. Define and compare NRZ-L and NRZ-I.
Both are Polar Non-Return to Zero encoding schemes. NRZ-L changes its voltage level whenever a different bit is encountered. NRZ-I changes its voltage level only when a bit '1' is encountered, which aids in signal synchronization.

12. Define Multiplexing. List its types.
Multiplexing allows multiple input signals to share a single common transmission medium, thereby avoiding collisions and making transmission cost-effective. Its main types are Frequency-Division Multiplexing (FDM), Wavelength-Division Multiplexing (WDM), and Time-Division Multiplexing (TDM).

13. What is the effect of noise on signal transmission?
Noise introduces unwanted electrical signals—such as thermal noise, crosstalk, or impulse spikes from power lines—into the transmission medium. These unwanted signals merge with the original transmission, causing the received signal to be distorted or corrupted.

14. Write the advantages of Differential Manchester Encoding.
Differential Manchester relies on an inversion in the middle of the bit to provide robust clock synchronization. Since it uses transitions rather than fixed voltage levels to represent logic (a transition means 0, no transition means 1), it reliably prevents DC component issues and handles noise better.

15. Mention the difference between TDM and FDM.
In Frequency-Division Multiplexing (FDM), multiple signals operate at the exact same time but utilize different, subdivided frequency channels. In Time-Division Multiplexing (TDM), all signals use the same frequency but operate at different times by taking turns in specific time slots.

16. List two characteristics of infrared transmission.
Infrared operates in the frequency range of 300 GHz to 400 THz and is strictly used for short-range, line-of-sight propagation. Because it cannot penetrate walls, it provides a high level of natural security confined to a single room.

17. You are given the following characteristics: Voltage changes in the middle of each bit, High-to-low transition for bit 1 and low-to-high for bit 0. Identify the encoding scheme used and justify your answer.
This describes Manchester Encoding. Manchester encoding utilizes a mandatory transition at the middle of each bit interval to represent logic states, combining both the data and the clock synchronization signal into a single transmitting wave.

18. Write any two applications of fiber optic cables.
Fiber-optic cables are widely utilized in major backbone networks (like SONET) because their extremely wide bandwidth makes them cost-effective over long distances. They are also heavily used by Cable TV companies and high-speed LANs, such as Fast Ethernet.

Unit 3: Data Link Control & MAC Protocols

Part A – Very Short Questions (1 Mark)

1. Define framing in data link control.
Framing is a process performed by the Data Link Layer that divides the continuous bitstream into manageable, formatted units called frames.

2. What is a single-bit error?
A single-bit error means that exactly one bit within a given data unit has been altered from a 1 to a 0, or from a 0 to a 1 during transmission.

3. Expand CRC.
CRC stands for Cyclic Redundancy Check, which is a mathematical error detection method utilizing polynomial binary division.

4. State the purpose of redundancy in error detection.
Redundancy involves appending extra, predictable bits to the data unit at the sender's side so the receiver can verify whether the data was corrupted during transit.

5. Define parity bit.
A parity bit is an extra redundancy bit added to a data unit (in Vertical Redundancy Check) to ensure that the total number of 1s in the unit is either consistently even or odd.

6. What is a MAC address?
A MAC (Media Access Control) address is a unique, physical hardware address assigned to a device's Network Interface Card (NIC) for local network communication.

7. List any one type of ARQ protocol.
One type of Automatic Repeat Request (ARQ) protocol is Stop-and-Wait ARQ.

8. Name the standard IEEE LAN protocol number for Ethernet.
The standard IEEE LAN protocol number for Ethernet is IEEE 802.3.

9. What is checksum?
Checksum is an error-detection technique where a data unit is divided into sections, added using one's complement, and the complemented sum is sent alongside the data.

10. What does ARQ stand for?
ARQ stands for Automatic Repeat Request, representing error control schemes that involve error detection and automatic retransmission of corrupted frames.

11. State one limitation of Stop-and-Wait ARQ.
Stop-and-Wait ARQ is highly inefficient because the sender is forced to pause and wait for an acknowledgment before sending the next frame, leading to poor link utilization.

12. What is the minimum size of an Ethernet frame?
The minimum size of a standard Ethernet frame is 64 bytes (including the MAC header, payload, and FCS).

13. Write the full form of CSMA/CD.
CSMA/CD stands for Carrier Sense Multiple Access with Collision Detection, which is the access method historically used by IEEE 802.3 Ethernet.

Part B – Short Answer Questions (2 Marks)

14. Differentiate between single-bit error and burst error.
A single-bit error corrupts only one bit in a data unit, which is highly unlikely in serial transmission. A burst error corrupts two or more bits in a data unit, which is much more common in serial transmission due to extended noise durations.

15. What is checksum? How does it work in error detection?
Checksum works by dividing the data unit into 'k' sections of 'n' bits. The sender adds all sections using one’s complement, complements the final sum to create the checksum, and sends it. The receiver repeats this process; if the final complemented result is zero, the data is accepted.

16. Explain forward error correction and backward error correction.
Forward error correction involves the receiver using an error-correcting code (like the Hamming code) to automatically pinpoint and reverse corrupted bits without needing the sender again. Backward error correction relies on the receiver asking the sender to retransmit the entire data unit via an ARQ protocol.

17. List advantages and disadvantages of error detection.
Error detection ensures high communication reliability by catching corrupted packets using redundancy checks (like VRC or CRC). However, simple methods like Checksum or VRC can easily fail to detect errors if an even number of bits are damaged in corresponding positions simultaneously.

18. Define flow control. List its two categories.
Flow control is a mechanism that ensures a fast-sending entity does not overwhelm a slower-receiving entity by preventing buffer overflows. Its two main categories are Stop-and-Wait flow control and Sliding Window flow control.

19. Explain Stop-and-Wait flow control with example.
In Stop-and-Wait, the source transmits a single frame and completely halts. It waits until the destination receives the frame and replies with an acknowledgment (ACK). Only after receiving the ACK does the sender transmit the next frame, which works well for a few large frames.

20. Differentiate between Go-Back-N ARQ and Selective Repeat ARQ.
In Go-Back-N ARQ, if a single frame fails, the receiver rejects it, and the sender must blindly retransmit the failed frame along with all subsequent frames. In Selective Repeat ARQ, only the specifically rejected frame is retransmitted, while the receiver intelligently buffers the subsequent good frames.

21. What is the minimum and maximum size of an Ethernet frame?
The IEEE 802.3 standard defines the minimum Ethernet frame size as 64 bytes and the maximum standard frame size as 1518 bytes.

22. Write short note on Unicast, Multicast, and Broadcast addresses.
A unicast address routes data directly from one sender to one specific receiver. A multicast address routes data to a specific, subscribed group of receivers. A broadcast address sends a data packet to absolutely every device currently connected to the local network.

23. State two advantages of error correction over error detection.
Error correction (like Hamming Code) mathematically pinpoints the exact location of a corrupted bit so the receiver can reverse it instantly. This eliminates the need for the sender to retransmit the data, saving significant bandwidth and reducing transmission delays.

24. List the fields of an IEEE 802.3 MAC frame.
While specific fields vary slightly by implementation, an Ethernet MAC frame typically consists of a Preamble, Start Frame Delimiter (SFD), Destination Address, Source Address, Length/Type field, Data Payload, and a Frame Check Sequence (CRC).

25. Define unicast, multicast, and broadcast with examples.
Unicast is one-to-one communication, such as an ARP reply sent exclusively back to the requesting router. Broadcast is one-to-all communication, like an initial ARP request sent to every computer on a LAN. Multicast is one-to-many communication directed at a designated subset of devices.

Unit 4: Routing, IP Networking & Security

Part A – Very Short Questions (1 Mark)

1. Define an IP address.
An IP (Internet Protocol) address is a unique string of numbers that serves as a logical address to identify every device connected to the internet or a local network.

2. Expand IPv4 and IPv6.
IPv4 stands for Internet Protocol version 4, and IPv6 stands for Internet Protocol version 6.

3. What is the size of an IPv4 address in bits?
An IPv4 address is exactly 32 bits in length.

4. Give one example of a private IPv4 address.
A very common example of a private IPv4 address is 192.168.1.1.

5. What is the range of Class A IP addresses?
The range of Class A IP addresses spans from 1.0.0.0 to 126.255.255.255, designed to support networks with a massive number of hosts.

6. Expand CIDR.
CIDR stands for Classless Inter-Domain Routing.

7. Define subnet mask.
A subnet mask is a number that extends the natural network mask by borrowing bits from the host ID portion to mathematically create a subnetwork ID.

8. What is the full form of DHCP?
DHCP stands for Dynamic Host Configuration Protocol.

9. What does ICMP stand for?
ICMP stands for Internet Control Message Protocol.

10. Name any one distance vector routing protocol.
Two practical implementations of distance vector routing protocols are RIP (Routing Information Protocol) and IGRP.

11. Which algorithm is used by Link State routing?
Link State routing protocols (like OSPF) use the Shortest Path First (SPF) algorithm, which is based on Dijkstra's algorithm.

12. Expand ARP.
ARP stands for Address Resolution Protocol.

13. State the purpose of NAT.
Network Address Translation (NAT) allows multiple devices on a private network to share a single public IP address to access the internet, mitigating the IPv4 shortage.

14. What does VPN stand for?
VPN stands for Virtual Private Network.

15. Define firewall in networking.
A firewall is a software program or hardware device that filters traffic entering a private network based on a common security policy to prevent unauthorized external access.

Part B – Short Answer Questions (2 Marks)

16. Differentiate between public IP address and private IP address.
Public IP addresses are globally unique and are assigned by ISPs to devices that directly access the internet. Private IP addresses are only unique locally within home or office networks and cannot communicate directly with the internet without NAT.

17. Write a short note on static vs dynamic IP addressing.
Static IP addresses are permanently assigned and are highly reliable for network servers requiring consistent access. Dynamic IP addresses are temporarily assigned from an available pool via DHCP, making them highly cost-effective for standard consumer devices.

18. State advantages of subnetting.
Subnetting allows network administrators to divide a massive network into smaller, logical subnetworks. This significantly increases routing efficiency, minimizes the size of the broadcast domain (reducing traffic congestion), and enhances overall network security.

19. Explain CIDR with an example.
CIDR (Classless Inter-Domain Routing) discards traditional rigid IP classes, instead splitting networks using a flexible prefix. For example, assigning 190.5.4.16/28 allocates exactly 14 usable IP addresses to a company, preventing address waste.

20. Differentiate between IPv4 and IPv6 (any four points).
1. IPv4 uses a 32-bit address; IPv6 uses a 128-bit address. 2. IPv4 uses decimal format (dots); IPv6 uses hexadecimal format (colons). 3. IPv4 has a variable header size (20-60 bytes); IPv6 has a fixed 40-byte header. 4. IPv4 utilizes broadcast transmissions; IPv6 abandons broadcast for multicast and anycast.

21. List the fields of an IPv4 datagram header.
An IPv4 datagram header contains vital routing information including the source address, destination address, and fields to manage packet fragmentation. Crucially, it also contains a Checksum field designed specifically to check the integrity of the header itself.

22. What is the role of ICMP in networking?
ICMP operates at the network layer to send vital control and error-reporting messages. It notifies senders of issues like "destination unreachable" and is the underlying protocol driving diagnostic utilities like Ping and Traceroute.

23. Explain ARP cache and ARP request.
An ARP cache is a local table where a sender temporarily stores recently resolved MAC addresses for future use. If an address isn't cached, the sender broadcasts an ARP request across the LAN, asking the device with a specific IP address to reply with its MAC address.

24. Differentiate between static and dynamic routing.
Static routing relies on manually configured routing tables that do not adapt to network changes. Dynamic routing uses automated algorithms (like Distance Vector or Link-State) to dynamically calculate the most efficient paths based on real-time factors like congestion and cost.

25. Write two advantages of link-state routing over distance vector routing.
Link-State routing possesses global knowledge of the entire network topology, allowing it to converge much faster than distance vector routing. Furthermore, it completely avoids the "count to infinity" and persistent looping problems that plague distance vector algorithms.

26. Define gateway of last resort.
A gateway of last resort (or default gateway) is a router that acts as a passage to connect a local network to external networks (like the internet). It is where data packets are sent when their specific destination IP address does not match any route in the local routing table.

27. Write short note on VPN.
A Virtual Private Network (VPN) creates a secure, encrypted tunnel over the public internet. It provides strong authentication, hides the user's real IP address to ensure privacy, bypasses geographic restrictions, and allows remote employees to safely access corporate resources.

28. State any two common network security threats.
Two severe network security threats include Man-in-the-Middle (MitM) attacks, where hackers intercept unsecured communications to steal data, and SQL Injections, which exploit database vulnerabilities to extract sensitive information.
